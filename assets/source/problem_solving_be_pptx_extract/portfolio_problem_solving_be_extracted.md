### Slide 44. Lessons Learned

- Lessons Learned
- 사내 코드 생성 AI 서비스 PoC 개발
- 개인용으로 개발된 go 기반 ollama는 상당히 느렸고 3명이 동시 요청 시에 throughput 감당을 하지 못하였다
- vLLM은 확실히 동시 접속에 대해서 ollama 보다 괜찮은 모습을 보였으나 CUDA 11.8과 호환이 되지 않아 직접 라이브러리의 소스 코드를 고쳐야 실행이 되었다
- Model serving
- Continue는 확실히 좋은 것 같고, 다양한 custom api endpoint를 만들어서
- Github copilot과 비슷한 경험을 원하면 추천할 수 있음
- Code Agent Extension
- 모델 크기와 성능은 log 비례하는 듯 하다
- 구조화 된 output을 딱 떨어지게 만드는 것은 매우 힘들다
- 한국어를 잘 못하길래, 영어로 된 많은 데이터 셋을 sLLM으로 번역 하여 fine-tuning 학습을 하려 했지만적절한 LLM을 찾는 과정도 고통이고, 그에 맞는 prompt를 디자인하는 것도 고역이었다.
- Prompt Engineering을 좀 더 편하게 할 수 있는 플랫폼/디자인이 있으면 좋을 것 같다
- LLM model
