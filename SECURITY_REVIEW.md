# 공개 전 보안 체크리스트

GitHub Pages는 공개 웹사이트로 노출될 수 있으므로, 아래 항목을 확인한 뒤 repository를 public으로 전환하거나 링크를 제출하세요.

## 민감 정보 제거

- [ ] 회사 내부 프로젝트 코드명, 내부 서비스명, 조직명 노출 여부 확인
- [ ] 내부 IP, 도메인, 포트, 방화벽 정책 등 인프라 세부 정보 제거
- [ ] 사내 시스템 구조가 과도하게 상세하게 드러나지 않는지 확인
- [ ] 장애 원인이 보안 취약점 악용에 가까운 형태로 읽히지 않는지 확인
- [ ] 미공개 특허, 미공개 서비스, 진행 중 프로젝트의 세부 구현을 공개해도 되는지 확인
- [ ] 고객사, 협력사, 내부 담당자명, 운영 수치 등 식별 가능한 정보 제거

## 표현 수위 조정

- [ ] “폐쇄망”, “방화벽”, “CVE” 등 민감하게 보일 수 있는 표현의 공개 가능 여부 확인
- [ ] 장애 대응 사례는 원인 분석 역량 중심으로 표현하고, 재현 가능한 공격 정보는 제거
- [ ] 미완성 프로젝트는 “진행 중”, “검토”, “설계” 범위를 명확히 표시
- [ ] 정량 수치는 외부 공개 가능한 범위인지 확인

## 권장 공개 방식

- 공개 이력서에는 포트폴리오 링크만 포함
- 상세 링크는 지원 회사에 제출하는 문서에만 포함
- 필요하다면 repository를 public으로 전환하기 전 별도 PDF 백업본을 만들어 보관
