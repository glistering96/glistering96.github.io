Case 01

자동차선 적부 최적화 Backend Service

Long-running AI inference를 위한 callback/queue 기반 백엔드 구조

Backend Problem

  • 모델 추론 시간이 입력 난도에 따라 5~20분으로 편차가 큼
  • 기존 사내 시스템의 요청/응답 흐름과 연결해야 함
  • Polling은 서버 자원 낭비가 크고, 동기 응답은 timeout 위험이 큼
  • Scale-out 시 요청을 받은 pod와 결과를 받는 pod가 달라질 수 있음

Key Design

  • API 서버와 engine 서버의 책임 분리
  • Request queue를 통한 요청 진입량 제어
  • Result store와 callback processor를 통한 비동기 응답
  • API/engine/외부 모델/기존 시스템을 연결하는 표준 데이터 모델 설계
Internal SystemAPI ServerRequest QueueEngine WorkerResult StoreCallback
ADR · 동기 API 대신 callback + queue를 선택한 이유

장시간 추론 작업에서 동기 응답은 timeout과 resource blocking 위험이 컸습니다. Queue를 도입해 엔진 서버 리소스를 보호하고, callback으로 기존 시스템에 결과를 반환하는 구조를 선택했습니다.

Case 02

Personal AI Agent Runtime Platform

사용자별 격리 runtime을 제공하는 Agent 실행 플랫폼

Backend Problem

  • Agent는 stateless API가 아니라 사용자별 workspace와 실행 상태가 필요함
  • 사용자별 파일, session, runtime, 로그가 섞이면 안 됨
  • 공용 skill은 중앙 관리하되 실행 환경은 격리해야 함
  • 오류 발생 시 사용자별 로그 추적이 가능해야 함

Key Design

  • Backend control layer가 runtime 상태를 관리
  • Kubernetes StatefulSet 기반 사용자별 runtime identity 구성
  • 공용 skill registry와 사용자별 workspace 분리
  • Playwright 기반 E2E 테스트 자동화 검토

ADR · Deployment 대신 StatefulSet을 고려한 이유

Agent runtime은 사용자별 고정 identity와 workspace 추적이 중요했습니다. Stateless workload 중심의 Deployment보다 StatefulSet이 사용자별 runtime 식별과 운영 추적에 적합하다고 판단했습니다.

Case 04

Steel TMS MLOps Backend

ML 추론과 영업소별 도메인 규칙을 결합한 자동배차 백엔드

Backend Problem

  • 5개 영업소의 배차 규칙이 서로 다름
  • 현업 인터뷰와 실제 배차 데이터가 불일치하는 경우가 존재
  • ML 예측값만으로는 공정배차 요구사항을 만족하기 어려움
  • 사용자 피드백을 빠르게 반영하고 안정적으로 배포해야 함

Key Design

  • 차량 유형을 target으로 하는 multi-class prediction 구성
  • 영업소별 요구사항을 Scorer 객체로 모듈화
  • 물량 N × 차량 M score matrix 계산 후 배차 수행
  • GitLab CI/CD, 로그 집약, retention, unit test 도입

Performance Note

Freight Matching API Optimization

3초 응답 제한을 만족하기 위한 복화 매칭 API 최적화

B2C 모바일 서비스에서 왕복 구간의 철강 물량을 빠르게 매칭해야 했습니다. 기존 로직은 50개 이상 entry에서 약 15분이 걸렸고, 단순 multiprocessing이나 chunking은 요구사항을 만족하지 못했습니다.

문제를 후보쌍 생성과 조건 필터링 문제로 재정의했습니다. 원본 테이블을 self-merge하여 후보쌍을 만든 뒤, 시간 조건과 불가능한 business logic을 boolean indexing으로 제거해 계산 시간을 약 900초에서 약 1초 수준으로 줄였습니다.

900s → 1spandas self-merge + boolean indexing
